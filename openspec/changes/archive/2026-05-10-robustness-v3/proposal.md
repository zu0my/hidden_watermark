# Robustness v3: 精准对齐 + 信号聚焦

## 背景

v2 引入的四层策略（A: 中心区域检测, C: PRN 校正, D: 块冗余, E: 多频段）取得了显著进展——旋转 10° 和缩放 95% 从不通过变为通过。但存在两个问题：

1. **信号稀释**：多频段 (E) 的加入使干净图像 ratio 从 26x 降至 2.09x。低频段的信号贡献不足以抵偿它在 blur/JPEG 攻击下引入的噪声。

2. **小角度旋转悖论**：旋转 10° 通过了 (1.96x) 但 1-5° 仍失败。根因是 PRN 校正的块级 (16px) 双线性插值对小偏移无效——当偏移量小于 1 个块时，权重退化为 w00≈1，校正失效。

## 目标

在不增加代码复杂度的前提下，通过三招精确击破每个剩余边界：

| 攻击 | v2 结果 | v3 目标 |
|------|---------|---------|
| 旋转 1-5° | -0.04x~0.67x | > 1.2x |
| 旋转 10-15° | 0.23x~1.96x | > 1.5x |
| JPEG q20 | 0.43x | > 1.0x |
| blur r5 | -0.82x | > 1.0x |
| 裁剪 75% | -0.24x | > 1.0x |
| 干净图像 | 2.09x | > 3.0x |

## 策略

**Step 1: 去除低频段，恢复信号聚焦**
- 删除多频段 (E) 的嵌入和检测代码
- 只保留中频段，信号不再分散

**Step 2: 混合对齐策略**
- ORB 返回旋转角度 → 路由选择
- `|角度| ≤ 3°` 且 `scale ≈ 1.0`：像素 warp + 传统检测（小偏移量下 warp 插值误差小）
- `|角度| > 3°` 或 `scale ≠ 1.0`：PRN 校正路径（当前已有）

**Step 3: 增强信号强度 + 块冗余**
- 嵌入强度提升到 1.5×（v2 已用 0.5，有 PSNR 余量）
- 块冗余从 3 块 (左/上) 扩展到十字形 5 块 (左/上/右/下)

## 非目标

- 不增加新依赖
- 不改变公共 API
- 不引入多频段
