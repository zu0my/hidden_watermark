# 统一 Warp 路径验证

## 假设

v3 的信号强度已从 baseline 的 26x 提升到 36x（1.4 倍）。在这个信号水平下，warp 的插值误差可能已不足以淹没水印信号。如果验证通过，整个架构可以从"双路由 + PRN 校正 + 双频段嵌入"简化为"统一 warp 路径 + 单频段嵌入"。

## 验证方法

在 v3 代码基础上，将 `detect_watermark` 中的路由逻辑改为：**所有场景统一走 warp 路径**。不修改嵌入逻辑。

跑摸底测试，对比数据。

## 成功判据

| 攻击 | 当前 v3 | 统一 warp 预期 |
|------|---------|---------------|
| 干净 | 36.55x | > 30x |
| JPEG q20 | 23.32x | > 15x |
| 旋转 1° | 2.84x | > 1.0x（关键） |
| 旋转 5° | 0.41x | > 1.0x（关键） |
| 旋转 10° | 1.20x | > 1.0x（关键） |
| 缩放 70% | 2.94x | > 1.0x |
| 裁剪 75% | 0.95x | > 1.0x |
| blur r3 | -0.25x | > 1.0x |

## 如果验证通过

- 删除 `detect_via_prn_correction`（~200 行）
- 嵌入端只保留中频段，PRN key 恢复为原始 `key`（不加 `_mid` 后缀）
- 路由逻辑删除

## 如果验证不通过

- 回退路由逻辑，回到 v3 状态
- 根据数据确定下一步修正方向
