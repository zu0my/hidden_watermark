# Robustness v2: 技术方案

## 整体架构变化

```
变化前 (当前)                             变化后
──────────────────────────────────      ──────────────────────────────────
suspect → warp → 切块 → DCT             suspect → 切块 → DCT
                                        original → 切块 → DCT
                                          ↓
                                        ORB 估计变换 T
                                          ↓
                                        PRN 域校正 (方案 C)
                                          每个块根据 T 选择 PRN 偏移
                                          ↓
                                        相关检测 (只取中心 50% 区域, 方案 A)
```

不再对 suspect 做像素 warp，完全消除插值误差。

## 方案 A: 中心区域检测

### 逻辑

在 `detect_watermark` 中，计算 score 和 threshold 时只使用图像中心区域的块：

```
┌──────────────────────┐
│  ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒  │
│  ▒  ┌──────────┐  ▒  │  ▒ = 边缘区域 (各 1/4，跳过)
│  ▒  │  检测区域  │  ▒  │
│  ▒  │  中心 50%  │  ▒  │
│  ▒  └──────────┘  ▒  │
│  ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒  │
└──────────────────────┘
```

### 影响
- 减少可用块数约 75% → threshold 升高（因为 n_blocks 变小）
- 但每个块的信号质量更高（无插值失真）
- 适用场景：所有几何攻击后的检测

## 方案 C: PRN 域校正

这是核心创新。不 warp suspect，而是调整 PRN 序列与 suspect DCT 块对齐。

### 原理

```
original 块网格:           suspect 块网格 (旋转 2° 后):
┌────┬────┬────┬────┐     ┌────┬────┬────┬────┐
│ B0 │ B1 │ B2 │ B3 │     │ ╲  │ ╲  │ ╲  │ ╲  │
├────┼────┼────┼────┤     │  ╲ │  ╲ │  ╲ │  ╲ │
│ B4 │ B5 │ B6 │ B7 │     ├────┼────┼────┼────┤
├────┼────┼────┼────┤     │  ╱ │  ╱ │  ╱ │  ╱ │
│ B8 │ B9 │B10 │B11 │     │ ╱  │ ╱  │ ╱  │ ╱  │
├────┼────┼────┼────┤     └────┴────┴────┴────┘
```

对于 suspect 的每个块 S_b，ORB 变换 T 告诉我们在 original 中它对应哪个区域：

```
T(S_b 的中心点) = (ox, oy)

在 original 块网格中找距离 (ox, oy) 最近的 4 个块:
  ┌──────┬──────┐
  │ B_i  │ B_j  │     ← 对应 4 个块的双线性权重
  ├──────┼──────┤
  │ B_k  │ B_l  │
  └──────┴──────┘

计算加权平均的 PRN 向量:
  PRN_eff[b] = w_i * PRN[B_i] + w_j * PRN[B_j] + w_k * PRN[B_k] + w_l * PRN[B_l]
```

### 检测逻辑变化

```
当前:
  score = Σ (DCT_suspect[b] - DCT_orig[b]) · PRN[b]

方案 C:
  // 对每个 suspect 块, 找到其在 original 中的对应位置
  for each suspect block b:
    (ox, oy) = T(block_center(b))     // 用 ORB 变换映射
    nearest = find_4_nearest_blocks(ox, oy) in original grid
    weights = bilinear_weights(ox, oy, nearest)
    diff = Σ w_k * (DCT_suspect[b] - DCT_orig[nearest[k]])
    score += diff · PRN[b]
```

## 方案 D: 块冗余嵌入

### 原理

每个嵌入位置由 3 个相邻块共享，检测时用"多数表决"方式计算信号：

```
嵌入时:
  比特 b_i → 嵌入到块 (bx, by), (bx+1, by), (bx, by+1) 的中频

检测时:
  score_i = score(bx, by) + score(bx+1, by) + score(bx, by+1)
  // 三个块中只要有一个对齐, 信号就能通过
```

### 影响
- 嵌入容量降低为原来的 1/3
- 嵌入时需确保 3 个块都存在（边界处理）
- 抗裁剪能力提升（只要三个块中有一个完整保留）

## 方案 E: 多频段嵌入

### 原理

当前只使用 DCT 中频（zigzag 8-24）。扩展到三个频段：

```
原:
  ┌─────┬─────┬─────┐
  │ DC  │ 低频 │     │
  ├─────┼─────┼─────┤
  │     │ 中频  │     │  ← 当前仅用此
  ├─────┼─────┼─────┤
  │     │     │ 高频 │
  └─────┴─────┴─────┘

新:
  低频段 (zigzag 3-7):  强度 0.3×strength, 对缩放鲁棒
  中频段 (zigzag 8-24): 强度 1.0×strength, 当前主力
  高频段 (zigzag 25-40): 强度 0.5×strength, 对旋转鲁棒
```

检测时三分段独立做相关，最终 score 加权合并。

## 四层组合的协同效应

```
                    ┌─────────────┐
                    │  方案 A     │  ← 始终生效（仅中心区域）
                    │  核心检测   │
                    └──────┬──────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                  ▼
   ┌──────────┐     ┌──────────┐      ┌──────────┐
   │ 方案 C   │     │ 方案 D   │      │ 方案 E   │
   │ PRN校正  │     │ 块冗余   │      │ 多频段   │
   └──────────┘     └──────────┘      └──────────┘
         │                 │                  │
         ▼                 ▼                  ▼
   消除插值误差        抗裁剪偏移         频率互补

   组合效果:
   旋转 1° → A(避开边缘) + C(PRN校正) + E(高频鲁棒) → 通过
   缩放 95% → A + C + E(低频鲁棒) → 通过
   裁剪 75% → A + D(块冗余) → 通过
```
